一条请求的完整流转（多 Agent 并行场景）
01 用户输入 → Orchestrator NLU 识别意图（多个则准备并行派发）
02 Orchestrator 构造 {task_id, domain, slots, lang_adapter, timeout_ms} 下发
03 专域 Agent 并行执行：子图检索 → 工具调用 → Policy 核验 → 置信度评估
04 各 Agent 返回 {confidence, response_draft, evidence, escalate_flag}
05 Orchestrator 合并结果 → 语言适配器生成最终回复
06 任一 Agent 触发 escalate_flag=true → Orchestrator 统一升级人工
